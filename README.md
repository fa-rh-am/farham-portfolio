# Farham — پرتفولیوی شخصی (رزومه آنلاین)

یک سایت پرتفولیوی تک‌صفحه‌ای با React + Vite + Tailwind CSS، شامل بخش‌های Home، Timeline (تایم‌لاین مسیر زندگی به‌شکل «مسیر کوهنوردی» با نشانگر متحرک)، Projects (با کارت + مودال جزئیات پروژه)، About/Skills و Contact. کاملاً واکنش‌گرا (Responsive) و روی موبایل تست‌شده.

---

## بخش ۱ — آموزش گام‌به‌گام اجرا در IntelliJ IDEA (از صفر)

### گام ۱: نصب Node.js

اول از همه باید Node.js نصب باشد (چون npm و کل ابزار React از طریق آن اجرا می‌شوند).

1. به آدرس `https://nodejs.org` بروید.
2. نسخه‌ی **LTS** را دانلود و نصب کنید (نسخه‌ی معمولی، نه Current).
3. بعد از نصب، یک ترمینال (Command Prompt یا PowerShell در ویندوز، Terminal در مک/لینوکس) باز کنید و بنویسید:

```bash
node -v
npm -v
```

اگر شماره نسخه نمایش داده شد، نصب موفق بوده.

### گام ۲: نصب IntelliJ IDEA

اگر از قبل نصب نکرده‌اید:

1. به `https://www.jetbrains.com/idea/download` بروید.
2. نسخه‌ی **Community** رایگان است و برای این پروژه کافی است (ولی برای پشتیبانی کامل از JSX/React، نسخه‌ی **Ultimate** تجربه‌ی بهتری می‌دهد چون پلاگین JavaScript/React توش از قبل هست؛ در نسخه‌ی Community هم می‌شود با نصب پلاگین کار کرد).
3. نصب را کامل کنید و IntelliJ را باز کنید.

> نکته: اگر IntelliJ Community دارید و منوی «Open» پروژه‌ی جاوااسکریپتی را کامل ساپورت نکرد، از پلاگین رایگان **"JavaScript and TypeScript"** در `Settings → Plugins` استفاده کنید (در Ultimate از پیش نصب است).

### گام ۳: باز کردن پروژه در IntelliJ

1. فایل zip تحویلی را در یک مسیر دلخواه (مثلاً روی Desktop) اکسترکت/Extract کنید. یک پوشه به اسم `portfolio` ساخته می‌شود.
2. IntelliJ IDEA را باز کنید.
3. از صفحه‌ی اول روی **Open** کلیک کنید.
4. پوشه‌ی `portfolio` را انتخاب و تأیید کنید.
5. چند ثانیه صبر کنید تا IntelliJ فایل‌ها را ایندکس کند.

### گام ۴: باز کردن ترمینال داخل IntelliJ

1. از پایین پنجره‌ی IntelliJ، تب **Terminal** را کلیک کنید (یا از منو: `View → Tool Windows → Terminal`).
2. یک خط فرمان داخل خود IntelliJ باز می‌شود که مسیرش همان پوشه‌ی پروژه است.

### گام ۵: نصب پکیج‌ها (اولین‌بار، فقط یک‌بار لازم است)

داخل همان ترمینال IntelliJ این دستور را بزنید و صبر کنید تمام شود (چند ده ثانیه طول می‌کشد):

```bash
npm install
```

این دستور تمام کتابخانه‌هایی که پروژه به آن‌ها نیاز دارد (React، Tailwind، framer-motion، lucide-react و ...) را دانلود و در پوشه‌ی `node_modules` نصب می‌کند.

### گام ۶: اجرای سایت به‌صورت لوکال (Development)

```bash
npm run dev
```

بعد از اجرا، خروجی مشابه این را می‌بینید:

```
  VITE v5.x.x  ready in xxx ms

  ➜  Local:   http://localhost:5173/
```

روی لینک `http://localhost:5173/` کلیک کنید (یا آن را در مرورگر کپی کنید) — سایت باز می‌شود. هر تغییری که در کد بدهید، بلافاصله در مرورگر به‌روزرسانی می‌شود (Hot Reload)، بدون نیاز به رفرش دستی.

برای توقف سرور، در ترمینال کلید `Ctrl + C` را بزنید.

### گام ۷: ساخت نسخه‌ی نهایی برای آپلود روی هاست (Build)

وقتی از نتیجه راضی بودید و خواستید سایت را آنلاین کنید:

```bash
npm run build
```

یک پوشه‌ی `dist` ساخته می‌شود. محتوای همین پوشه (نه کل پروژه) را روی هر هاستینگ استاتیک آپلود می‌کنید:

- **Vercel** یا **Netlify**: می‌توانید مستقیماً کل پوشه‌ی پروژه (نه فقط dist) را به گیت‌هاب push کنید و آن سرویس‌ها خودشان `npm install && npm run build` را اجرا می‌کنند.
- **GitHub Pages / Cloudflare Pages / هر هاست ساده**: فقط محتوای پوشه‌ی `dist` را آپلود کنید.

برای پیش‌نمایش همان build نهایی روی سیستم خودتان، قبل از آپلود:

```bash
npm run preview
```

## بخش ۲ — شخصی‌سازی محتوا (بدون نیاز به دست‌زدن به کدِ کامپوننت‌ها)

تقریباً همه‌ی متن‌ها، لینک‌ها و اطلاعات از یک فایل واحد خوانده می‌شوند:

```
src/data/content.js
```

در این فایل می‌توانید ویرایش کنید:

- `profile` → نام، لقب/هندل، نقش، سه‌ویژگی بالای هدر، مسیر عکس، محل زندگی، متن راهنمای گوشه‌ی صفحه
- `timeline` → مراحل تایم‌لاین (تاریخ، عنوان، توضیح، عکس اختیاری برای هر مرحله) — هر مرحله‌ی جدید که اضافه کنید، مسیر SVG و نقطه‌ها به‌طور خودکار خودشان را با تعداد جدید تنظیم می‌کنند
- `skills` → مهارت‌ها به تفکیک دسته (زبان‌ها، فرانت‌اند، بک‌اند، ابزارها)
- `projects` → کارت‌های پروژه؛ هر پروژه شامل عنوان، سال، خلاصه، سه نکته‌ی برجسته (`highlights`) که در پنجره‌ی جزئیات نمایش داده می‌شوند، تگ‌های تکنولوژی، و لینک گیت‌هاب یا سایت زنده
- `contact` → ایمیل، پیام کوتاه و لینک‌های شبکه‌های اجتماعی

فایل را با هر ادیتوری (حتی Notepad) می‌شود باز کرد، ولی داخل IntelliJ راحت‌تر است چون syntax highlighting دارد.

## اضافه‌کردن عکس واقعی خودتان

عکس‌ها را داخل پوشه‌ی `public/` بگذارید (مثلاً `public/photo-hero.jpg`) و مسیر آن را در `content.js` در فیلد `profile.photo` وارد کنید (مثلاً `/photo-hero.jpg`). اگر عکسی در آن مسیر پیدا نشود، به‌طور خودکار یک تصویر SVG کوهستانی جایگزین (`ScenicArt`) نمایش داده می‌شود؛ یعنی سایت هیچ‌وقت با آیکون عکس شکسته دیده نمی‌شود. همین قانون برای عکس هر مرحله از تایم‌لاین هم صدق می‌کند (فیلد `photo` در هر آیتم `timeline`).

## ناوبری سایت

- کلیک روی آیتم‌های نوار کناری (دسکتاپ) یا نوار پایین صفحه (موبایل) → پرش مستقیم به هر بخش
- کلیدهای `↑` / `↓` یا `Space` → حرکت بین بخش‌های اصلی سایت
- داخل بخش Timeline: کلیدهای `←` / `→` یا کلیک روی نقطه‌های مسیر → حرکت بین مراحل زندگی؛ یک نشانگر «رد پا» روی مسیر با نقطه‌ی فعال حرکت می‌کند و در آخرین مرحله جشن کوچکی (ذرات رنگی) نمایش داده می‌شود
- داخل بخش Projects: کلیک روی هر کارت → باز شدن یک پنجره‌ی جزئیات با توضیح کامل‌تر، نکات برجسته و لینک مستقیم؛ با `Esc` یا کلیک بیرون پنجره بسته می‌شود

## واکنش‌گرا بودن (Responsive)

سایت روی سه اندازه تست شده: موبایل (حدود ۳۹۰px عرض)، تبلت و دسکتاپ.

- نوار ناوبری در موبایل به یک نوار افقی شناور پایین صفحه تبدیل می‌شود (فقط آیکون).
- اندازه‌ی فونت عنوان اصلی و فاصله‌گذاری بخش‌ها به‌صورت پلکانی (`text-5xl → sm:text-7xl → md:text-8xl`) کوچک‌تر می‌شود تا در گوشی‌های کوچک هم زیبا و بدون سرریز (overflow) بماند.
- کارت‌های پروژه در موبایل تک‌ستونه و در صفحات بزرگ‌تر دوستونه می‌شوند.
- پدینگ پایین هر بخش در موبایل بیشتر است تا نوار ناوبری شناور روی محتوا نیفتد.

```
├── public/

└── src/
    ├── main.jsx                نقطه‌ی ورود React
    ├── App.jsx                 چیدمان کلی + مدیریت ناوبری بین بخش‌ها
    ├── index.css                استایل‌های پایه، اسکرول‌اسنپ، اسکرول‌بار
    ├── data/
    │   └── content.js           همه‌ی محتوای متنی/لینک‌ها — اینجا را ویرایش کنید
    ├── hooks/
    │   └── useSectionNav.js     مدیریت اسکرول‌اسنپ + ناوبری با کیبورد
    └── components/
        ├── Sidebar.jsx           نوار ناوبری (دسکتاپ کناری / موبایل پایین)
        ├── Hero.jsx               بخش معرفی اول صفحه
        ├── Timeline.jsx           بخش «مسیر زندگی» — کامپوننت شاخص سایت
        ├── Projects.jsx           گرید پروژه‌ها + پنجره‌ی جزئیات
        ├── About.jsx              درباره‌ی من + مهارت‌ها
        ├── Contact.jsx            بخش ارتباط
        ├── ScenicArt.jsx          تصویر جایگزینِ SVG کوهستانی (وقتی عکس واقعی نیست)
        └── ParticleField.jsx      ذرات شناور تزئینی در پس‌زمینه (کم‌رنگ و آرام)
```

## پالت رنگی

طیف آرام و کم‌کنتراست از آبی، سبز، زرد و کِرِم — همه در `tailwind.config.js` به‌صورت توکن‌های `cream` / `ink` / `sky` / `sage` / `gold` تعریف شده‌اند تا تغییر رنگ کل سایت فقط با ویرایش همان چند مقدار hex ممکن باشد.

## نکات فنی

- انیمیشن‌ها با `framer-motion` ساخته شده‌اند و برای کاربرانی که `prefers-reduced-motion` را فعال کرده‌اند به‌طور خودکار محدود می‌شوند.
- آیکون‌ها از `lucide-react`.
- استایل‌دهی با Tailwind CSS.
- خروجی `npm run build` تست شده و بدون خطا کامپایل می‌شود.

## مشکل رایج و راه‌حل

- **پورت 5173 قبلاً استفاده شده**: اگر خطای «Port already in use» دیدید، یا برنامه‌ی قبلی را ببندید یا Vite به‌صورت خودکار پورت بعدی (5174 و ...) را انتخاب می‌کند.
- **`npm install` خطا داد**: مطمئن شوید Node.js نسخه‌ی ۱۸ به بالا نصب است (`node -v`).
- **صفحه سفید است**: کنسول مرورگر را با `F12` باز کنید و پیام خطا را بخوانید؛ معمولاً به دلیل اشتباه تایپی در `content.js` است.
